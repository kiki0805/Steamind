About

Steamind is a tool for visualizing games from your own Steam library as well as other games from Steam that might suit your interests. It is the final project for the course DH2321 Information Visualization given at KTH Royal Institute of Technology. Initially inpsired by Steam's own recommendation system, we wanted to explore a different way of visualizing the data. By displaying the games that you own, Steamind aims to provide a way for you to gain insight of your own gaming preferences such as which categories you prefer, or help you come to terms with the uncomfortable thruth about your bundle-spending habits. Apart from that, Steamind also visualizes a selected set of other games that you might not own in hopes that you can use the visualizaiton to expand your library with games you might want to play.

The visualization consists of a force-directed graph organized in a hierarchical structure with two levels. Starting at the root node is the user and four branches representing game categories emerges from it. At the last level, we have the leaves which represent the games. The games are organized into these four main categories based on a research article from 2020 by Xiaozhou Li and Boyang Zhang [1].

The Data

The data was retrieved through Steam's API and through web-scraping. The nature of data is nominal, numerical, and categorical. The data is stored in a database through SQLite.

Learning Objectives

Our main learning objectives for this project was to learn to work with Javascript and Javascript libraries. We used d3.js and jQuery. We also created our own API which we used to gather, and filter, the data dynamically. This was done with Python, Scrapy, Flask, and Peewee.

[1] Li, X. & Zhang, B. 2020. A preliminary network analysis on Steam game tags: Another way of understanding game genres
